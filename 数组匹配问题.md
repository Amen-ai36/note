# 数组匹配问题（区间-点最大匹配）

## 0. 题目原文
给定两个整数数组 `q_1, q_2, ..., q_n` 和 `r_1, r_2, ..., r_n`，以及一个整数 `k`。

你可以执行任意次操作（也可以 0 次）：

1. 选择两个整数 `x, y`，满足：
   - `1 <= y < x <= k`
   - 存在下标 `i` 使得 `q_i = floor(x / y)`
   - 存在下标 `j` 使得 `r_j = x mod y`
2. 从数组 `q` 中删除一个 `q_i`，从数组 `r` 中删除一个 `r_j`（若有多个相同值，只删除一个）。

求最多能执行多少次操作。

---

## 1. 问题抽象
很多题目表面是“从数组 A 取一个、从数组 B 取一个并删除”，本质可以转成：

- A 中每个元素 `a_i` 对应一个可接受区间 `[L_i, U_i]`；
- B 中每个元素 `b_j` 是一个点；
- 一次操作 = 用一个点 `b_j` 去匹配一个区间 `[L_i, U_i]`，且每个区间和每个点最多使用一次；
- 目标：最大匹配数。

这是经典贪心模型：**区间-点最大匹配**。

---

## 2. 这道题如何转成区间
题意条件：

- `1 <= y < x <= k`
- `q = floor(x / y)`
- `r = x % y`

由整除定义：

- `x = q*y + r`
- `0 <= r < y`

对固定 `q`，问哪些 `r` 可行。

### 2.1 上界推导
因为 `r < y`，最小可取 `y = r + 1`。
代入 `x = q*y + r` 得最小 `x`：

`x_min = q*(r+1) + r = (q+1)r + q`

要可行，至少要 `x_min <= k`：

`(q+1)r + q <= k`

所以

`r <= floor((k-q)/(q+1))`

记

`U = floor((k-q)/(q+1))`

### 2.2 下界推导
一般 `%` 余数满足 `r >= 0`，故下界先是 `0`。

但当 `q=1` 时，如果 `r=0`，则 `x = y`，与 `y < x` 冲突。
所以 `q=1` 时必须 `r >= 1`。

记

- `L = 0`（默认）
- 若 `q == 1`，`L = 1`

### 2.3 无效 q
- `q <= 0` 无效（`x>y` 时 `floor(x/y) >= 1`）；
- `q > k` 无效（因为 `x<=k` 且 `q=floor(x/y)` 不可能太大）；
- 若 `U < L`，该区间为空，也无效。

于是每个 `q_i` 都变成一个区间 `[L_i, U_i]`，`r_j` 是点，问题完成转化。

---

## 3. 贪心策略（最优）
步骤：

1. 把所有有效区间按 `U`（右端点）从小到大排序；
2. 用 `multiset` 维护还未使用的 `r`；
3. 依次处理每个区间 `[L,U]`：
   - 找集合里最小的 `r >= L`（`lower_bound(L)`）；
   - 若该 `r <= U`，就匹配成功并删除它。

为什么对：

- 按最小右端点优先，避免“紧区间”被大点占掉；
- 每次选当前可用的最小点，给后面的区间留下更大空间。

这是标准交换论证可证最优的贪心。

---

## 4. 复杂度
- 排序区间：`O(n log n)`
- 每个区间一次 `lower_bound + erase`：`O(log n)`
- 总复杂度：`O(n log n)`
- 空间：`O(n)`

---

## 5. 通用代码模板（C++）

```cpp
// intervals: pair<U, L>
sort(intervals.begin(), intervals.end());
multiset<long long> S(points.begin(), points.end());

int ans = 0;
for (auto [U, L] : intervals) {
    auto it = S.lower_bound(L);
    if (it != S.end() && *it <= U) {
        ++ans;
        S.erase(it);
    }
}
```

---

## 6. 这题易错点
- `vector` 要先开大小再按下标读入：`vector<ll> q(n), r(n);`
- `q=1` 时要单独处理 `r=0` 不合法。
- 贪心里排序关键字是区间右端点 `U`，不是左端点。

---

## 7. 识别信号（以后看到就想到这个模型）
当题目出现这些特征时，优先考虑“区间-点匹配”：

- 两个数组中各取一个并删除；
- 可行性由一个值落在某范围内决定；
- 目标是最多配对数。

关键词：**最大可配对数、每个元素最多一次、满足区间约束**。
